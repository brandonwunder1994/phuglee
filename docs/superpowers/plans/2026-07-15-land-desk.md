# Land Desk (Home Vault / Land Vault) Implementation Plan — Phase 0–1

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Split land out of `/vault` (rename **Home Vault**) into `/land-vault` (**Land Vault** / Land Desk), keep Analyze→catalog publish unchanged, and ship Phase 1 MVP: 7-check KEEP/TOSS + fund match on existing `leadType=land` leads.

**Architecture:** Shared `data/leads-catalog/` stays. API gains `surface=home|land` so Home never lists/counts land. New Land Vault page clones Vault desk patterns. New `lib/leads-platform/land/` modules persist `landScreen` + `fundMatches` on the lead. Analyzer `vacant`/`vacant_lot` → `leadType=land` publish path is **not** rewritten.

**Tech Stack:** Node.js (`lib/leads-platform/`), vanilla `public/js/*`, `node:test`, fund catalog JSON, existing Max/auth roles.

**Spec (LOCKED):** `docs/superpowers/specs/2026-07-15-land-desk-design.md`

**Out of this plan (follow-on plans after Phase 0–1 verified):** Phase 2 LAO calculator, Phase 3 Land Comp engine, Phase 4 teardown bridge, Phase 5 Filter→Land preferential.

## Global Constraints

- Never wipe `data/leads-catalog/`, `data/filter-lists/`, `data/bridge-brain/`, or analyzer `users/` sessions.
- Do **not** change Analyze `publish-from-analyzer` / `vaultLeadTypeFromResult` mapping (`vacant`/`vacant_lot` → `land`).
- Do **not** create a second catalog store.
- Home Vault = `distressed` + `well_maintained` only; Land Vault = `land` only.
- Server-side `surface` filter required — removing the Land tab alone is insufficient.
- Vault-only user (`matt`) must be allowed on **both** `/vault` and `/land-vault`.
- Dispos user (`brad`) should get Land Vault too (same Max lead inventory access as Home Vault).
- After UI edits: `scripts/verify-live.ps1` + scoped `verify-mobile.ps1 -Pages "/vault,/land-vault"` before claiming live.
- Never claim Home/Land Vault “live” without proof in the same turn.
- `npm test` / targeted `node --test` green after each task.
- Product copy: **Home Vault**, **Land Vault**, in-page **Land Desk**.

**Interfaces:**

- Rules:
  - `surface=home` → allow only `leadType` ∈ {`distressed`,`well_maintained`}; if caller passes `leadType=land`, result empty

**Files:**

- Modify: `lib/leads-platform/api.js` — `GET/PUT /api/leads/:id/land-screen`
- Modify: `tests/land-screen.test.js`

**Interfaces:**
- Produces: `recommendLandVerdict(landScreen) → 'keep'|'toss'`
  - TOSS if `demandBuilders.status === 'fail'` OR any of checks 1–7 `status === 'fail'`
  - KEEP only if demand is `pass` and all seven are `pass`
  - If any `unknown` and no fails → still recommend `toss`? **Locked:** recommend `toss` only on hard fail; if unknowns remain with no fails → `recommendedVerdict: null` (needs screen). Operator may still set KEEP with `verdictNote` required when overriding a fail.
- Produces: `PUT` saves body, sets `recommendedVerdict`, `screenedAt`, `screenedBy` from session user
