# Requirements: Property Distress Analyzer — v1.3 Calm UI

**Defined:** 2026-06-30  
**Core Value:** Upload → scan → trust tiers → export/review distressed leads without UI clutter

## v1.3 Requirements

### Design System

- [ ] **DS-01**: Replace cyber-HUD color palette with calm neutral tokens (background, foreground, muted, accent, destructive)
- [ ] **DS-02**: Typography uses 3–4 sizes and 2 weights; distinctive but readable pairing (not Inter/Roboto)
- [ ] **DS-03**: Tailwind CLI build integrated; shadcn-compatible CSS variables on `:root`
- [ ] **DS-04**: Spacing on 4px grid; border-radius consistent (8–12px, not mixed neon glow radii)
- [ ] **DS-05**: Motion restrained — no blink/pulse/glow animations except scan progress indicator

### App Shell

- [x] **SHELL-01**: Remove fixed HUD bar (clock, "Distress Intelligence" ticker); status moves to command bar
- [x] **SHELL-02**: Sidebar reduced to 3 primary nav items + single overflow; admin submenus demoted
- [x] **SHELL-03**: Command bar shows: title, file status, save indicator, Start/Stop only
- [x] **SHELL-04**: Settings, AI Brain, backup, export, API tests reachable via ⌘K or overflow menu
- [x] **SHELL-05**: "DistressOS Intelligence Suite" branding replaced with calm product naming

### Workflow Surfaces

- [ ] **FLOW-01**: Empty state = one primary CTA + one secondary; tertiary actions in overflow/⌘K
- [ ] **FLOW-02**: Scan progress = slim inline bar; agent grid collapsed by default during scan
- [ ] **FLOW-03**: Summary KPIs show 3 primary metrics; secondary buckets in expandable breakdown
- [ ] **FLOW-04**: Live tier alerts demoted to subtle toast stack (not persistent card stack)
- [ ] **FLOW-05**: Log panel hidden by default; expandable for power users

### Results & Data Views

- [x] **DATA-01**: Filter toolbar consolidated to segmented control (3–4 tiers) + overflow for rest
- [x] **DATA-02**: Search field prominent; sort label subtle
- [x] **DATA-03**: Lead cards use calm hierarchy: address → tier badge → thumb; less chrome per card
- [x] **DATA-04**: Bulk edit behind explicit "Edit" mode toggle (not always-visible bar)
- [x] **DATA-05**: Table view matches card calm styling

### Modals & Review

- [x] **MODAL-01**: Settings/Upload/Brain modals use calm dialog pattern (native `<dialog>` or equivalent)
- [x] **MODAL-02**: Property inspector simplified — imagery-first, metadata secondary
- [x] **MODAL-03**: Review mode overlay calm dark scrim; action bar uses clear tier colors without emoji overload
- [x] **MODAL-04**: Score edit modal matches tier picker calm pattern

### Quality & Safety

- [x] **QA-01**: `npm test` passes (78+ tests) after every phase
- [x] **QA-02**: No regression in save/tier/backup/review-mode functionality
- [x] **QA-03**: Virtual scroll + progressive render performance unchanged at 10k leads
- [x] **QA-04**: All existing DOM IDs preserved or migration shim documented per phase

## Future Requirements (post v1.3)

- **THEME-01**: Light/dark mode toggle
- **A11Y-01**: Full WCAG audit pass
- **MOB-01**: Mobile-optimized review mode

## Out of Scope

| Feature | Reason |
|---------|--------|
| React/SPA migration | Vanilla shell works; UI is reskin + structure, not rewrite |
| Paid UI kits | Free stack constraint |
| Backend tier/save changes | v1.2 bones are complete and tested |
| New AI/classification features | UI milestone only |
| shadcn React components | Port tokens/patterns only |
