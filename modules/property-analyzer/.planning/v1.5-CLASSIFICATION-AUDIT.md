## 2. Research Insights

### What experienced investors look for (D4D / wholesaling)

**Strong distress signals (any one = stop):**
- Boarded openings, tarped roof, structural collapse, fire damage
- Junk/debris fields covering yard, not a single trash can
- Multiple broken windows + neglect context
- Abandoned vehicles + yard trash (not a daily driver in driveway)
- Combination: overgrown + peeling + deferred (dump-house pattern)

**Well-maintained signals:**
- Mowed/tidy yard, intact windows, maintained roof plane
- Single cosmetic issue (one gutter, light grass strip) on otherwise normal facade
- Occupied cues: curtains, maintained walkway, no debris accumulation

**Vacant land:**
- No roof polygon on subject parcel; weeds/junk on bare land ≠ distressed home
- Driveway alone insufficient

**Blurred / unusable:**
- Google privacy blur, extreme distance, obstruction, not "I can't tell if distressed"

### What works in modern AI approaches

1. **Separate percepts from policy** — Vision model lists observable facts; code applies tier policy (reduces subjectivity)
2. **Multi-view fusion** — Street facade + aerial roof/yard improves both distress and land/home accuracy (HUD, academic parcel studies, commercial providers like Cape/CAPE-style roof+yard)
3. **Calibrated confidence routing** — High confidence auto-accept; borderline → review (human-in-the-loop scaling)
4. **Golden-set regression** — Replay fixed image sets when prompts/rules change; track precision/recall per tier
5. **Asymmetric error costs** — For lead gen, false negative (missed distress) often costlier than false positive → policy should not default to "maintained" on ambiguity

## 3. Recommended Upgrades

### A. Wire dual-source pipeline (highest ROI)

**Change:** In `finalizeStreetAnalysis`, parallel-fetch satellite (from cache or API), run `classifyWithSatellite`, then `reconcileSatelliteWithStreetView(analysis, satelliteResult)`.

**Files:** `public/js/app.js` (primary), optionally `routes/maps.js` for combined endpoint

**Why:** Activates existing `fuseStreetAndAerialScore`, land/home conflict detection, manicured exemption with real roof/yard data.

### B. Split vision from decision in prompts

**Phase 1 prompt (observation):** "List visible indicators with certainty; do not assign final tier."

**Phase 2 (policy — existing tier engine):** Keep `computeLeadTier` as source of truth.

Or lighter lift: Remove "prefer well_maintained when uncertain" and replace with "report all moderate signals even on otherwise maintained homes."

**Files:** `public/js/config.js` (`buildD4DIndicatorGuide`, `buildD4DStreetScoringGuide`, `buildStaticTierRules`)

### C. Rescope downgrade rule

**Current:** `score >= 6 && !looksVisuallyDistressed` → well_maintained

**Proposed:** Downgrade only when `qualifiesManicuredExemption` OR (`confidence >= 70` AND aerial ≤ 5 AND soft indicators only). Otherwise route to tier review or keep distressed.

**Files:** `lib/tier-engine.js`, tests/fixtures/tier-cases.json

### D. Composite confidence score

```
classificationConfidence =
  min(streetConfidence, satelliteConfidence)  // when both exist
  - penalty for landHomeConflict
  - penalty for qualityFlags (stale_streetview, approximate_geocode)
  - penalty for indicator disagreement (AI distress language vs no indicators)
```

**Store:** `record.classificationConfidence` (0-100)

### E. Expand review routing (keep existing pages)

| Queue | Entry condition | Review mode |
|-------|-----------------|-------------|
| Needs Review (existing) | land/home conflict | `openReviewMode('needs_review')` |
| Tier Review (existing) | manual / sample | `openReviewMode('tier')` |
| **Low Confidence (new)** | confidence < 65 OR signal conflict | tier review lane with badge |
| **Borderline Distress (new)** | score 5-7 + 1 moderate indicator | prioritize in spot-check |

Review pages unchanged structurally — add filters and queue priority.

## 4. Blurred Images & Land — Specific Handling

### Blurred

| Current | Proposed |
|---------|----------|
| Fetch error → `buildNeedsReviewResult` → blurred | Pre-classify image quality before Gemini (blur/obstruction heuristic or Gemini quality field) |
| `unavailable` non-conflict treated as blurred | Split `imageryQuality: ok|degraded|unusable` from `category` |
| User review → `finalizeBlurredLead` | Keep; add retry queue for transient Gemini failures vs true blur |

### Vacant land

| Current | Proposed |
|---------|----------|
| Satellite vacant skip only on SV-absent path | Always run satellite structure check on first scan |
| SV says home, sat says vacant → unavailable review | Keep; lower threshold to 50% when geocode quality poor |
| Weeds on lot → AI sometimes scores as property | Strengthen vacant_lot post-check: if aerial score 0 and no structure_on_lot, force vacant regardless of street score |

## 5. Confidence Scoring & Review Routing

### Proposed thresholds (tunable via review data)

| Confidence | Tier action | Review |
|------------|-------------|--------|
| ≥ 85 | Auto-accept tier | Skip |
| 65-84 | Auto-accept with `lowConfidence` flag | Spot-check sample 5% |
| 45-64 | Auto-accept distressed only if HIGH indicator; else review | Tier review queue |
| < 45 | No auto tier | Needs review or blurred |

### Use review corrections as ground truth

Track from existing `tierCorrections` / `categoryCorrections`:
- `false_negative_rate`: distressed → well_maintained corrections
- `false_positive_rate`: well_maintained → distressed corrections

Tune thresholds to balance (default: favor catching distress).
