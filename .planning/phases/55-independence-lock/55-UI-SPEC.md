## Copywriting Contract

Independence messaging only. Phase 56 owns full Process → Train → Save → Download teaching pack.

| Element | Copy |
|---------|------|
| Primary CTA | **Save list** (existing `#bridge-save-list` — keep label; do not rename to Analyze verbs) |
| Forbidden CTAs | **Never show:** “Send to Analyze”, “Push to Analyze”, “Import to Analyzer”, “Open in Analyze”, or any control that POSTs `/api/bridge-import-records` or calls push helpers |
| Process success (no discards of interest) | `Processing complete. Save the list below — nothing was sent to Analyze.` |
| Process success (discards present) | `{N} row(s) discarded.` + optional distress / review fragments — **omit** Analyze-hidden fragment when `alreadyImported === 0` |
| Empty state heading | (lists) `No saved lists yet.` — keep |
| Empty state body | `Process a city file above, then click Save list.` — keep (Phase 56 may expand) |
| Error state (generic process) | Server `error` string + actionable next step already in `fetchJson` / `showError` |
| Error `NO_USABLE_ROWS` (default path) | Prefer distress / address / duplicate breakdown. **Do not** lead with “already in Analyze” when filter is off and counts are 0. |
| Error `NO_USABLE_ROWS` (opt-in only; rare) | Only if `alreadyImported > 0` under future opt-in: include `N already in Analyze` in breakdown — **not** default product path |
| Destructive confirmation | None in Phase 55 (no new destructive flows). List delete remains existing Filter behavior; out of redesign scope |
| Save success flash | `Saved “{name}”. Upload the next city file when ready.` — keep |
| Save empty guard | `Process a file with kept rows before saving.` — keep |

### Independence strings — locked allow / deny

| Context | Allowed (Phase 55) | Forbidden |
|---------|--------------------|-----------|
| Process stub note | “nothing was sent to Analyze” (reassurance) | “sent to Analyze”, “queued for Analyze”, “pushed” |
| Hidden-row note | Only if `stats.alreadyImported > 0` (opt-in filter on): `N already in Analyze (hidden from this list).` | Showing that sentence when count is 0; implying default hard-drop |
| Results meta suffix | Only if `importIndexCount > 0` **and** filter actually ran: `· N address(es) in Analyze` | Showing index count when index load was skipped (default) |
| KPI label | May retain **Already in Analyze** only when value > 0; otherwise omit the card (see Interaction) | Presenting 0 as “already removed” / implying purge emptied the list |
| Server `onlyImported` message | Engine may keep helper for opt-in; **must not** be the default re-work empty-state after Phase 55 | Default re-filter/purge path returning “Every record… already in your Analyze session” |

### `already_imported` off-by-default — display contract

When process runs with default policy (`applyAlreadyImportedFilter !== true`):

1. `stats.alreadyImported` is **0** — UI must not invent a non-zero “removed” count.
2. **Do not** show “already in Analyze (hidden from this list)” in stub note.
3. **Do not** append `· N address(es) in Analyze` from `processingMeta.importIndexCount` when count is 0 / index was skipped (target: 0 when off).
4. KPI grid: **omit** the “Already in Analyze” card when value is 0 (or keep grid layout but never phrase zero as “removed”). Prefer omit — honesty over a permanent dead KPI.
5. `NO_USABLE_ROWS` breakdown must not list “already in Analyze” unless the response stats actually include a positive already-imported count.
6. Full kept lists after re-work/purge are **correct** — no UI copy suggesting rows “should have been filtered because they are in Analyze.”

When future opt-in is true (engine-only this phase; **no UI toggle in 55**): existing positive-count strings may render. Do not ship a Filter settings toggle this phase.

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Process complete | Show results + Save panel for kept rows; never auto-navigate to Analyze; never auto-POST import |
| Save list | Explicit user click only; success flash on lists panel; no Analyze side effect |
| Train Approve/Deny | Admin-only; mutates brain + client row lists only; no Analyze write |
| Forbidden control | Any clickable that would push/import to Analyze — **not present**; if discovered, delete in same phase as adapter purge |
| Opt-in import filter | Engine flag only (`applyAlreadyImportedFilter === true`); **no** Filter UI checkbox in Phase 55 |
| Re-work / re-filter same city | Full kept list expected by default; UI must not explain emptiness as “already imported” unless opt-in was used |
