<!-- Working-set model (preserve) -->
```
lastResult (client memory only)
  processUpload → overwrites (AFTER dirty-guard)
  Train decision → mutates rows / groups / stats
  Save → POST /api/bridge/lists → resetImportAreaAfterSave
  Download → from list store (durable), not lastResult
```

<!-- Dirty-guard — exact UI-SPEC copy -->
When `lastResult?.rows?.length > 0` at start of `processUpload()` (after city/type/files checks, before spinner/fetch):
```
You have {N} kept row(s) that are not saved yet.

Process a new file anyway? Unsaved work (including any Train decisions) will be lost.
```
Native `window.confirm`. Cancel → return (do not process). OK → continue existing process path.
Use `n.toLocaleString()` for N. **Do not** auto-save to avoid the dialog.
Optional `beforeunload` is discretionary — not required.

<!-- Soft Train-before-Save — exact UI-SPEC copy; admin only -->
At start of `saveCurrentList()` after kept-rows guard, when `isBridgeAdmin()` and open undecided groups exist while Train mode is active (`resultsMode === 'train'`):
```
{N} Train group(s) are still visible. Save this list now?

Tip: Finish Approve/Deny first so this download matches your decisions.
```
Native confirm. Cancel → return without POST. OK → continue save.
Use existing `filterUndecidedTrainGroups` + `getReviewGroups(lastResult)` for N.
Customers / non-admin: never show. Never hard-block Save without confirm option.

<!-- Save flash — exact UI-SPEC -->
`Saved “{name}”. Upload the next city when ready — or download from Saved lists for enrichment.`
(fallback without name may mirror structure)

<!-- Post-decision Train status (admin) -->
After successful decision: include save nudge, e.g. core UI-SPEC line:
`Decision saved to brain. Save list below when this city is ready.`
May combine with remaining-group count; must mention Save list.

Do NOT: auto-save on process, auto-delete after download, call clearAllLists, wipe filter-lists, change export schema, rebuild list store.
</interfaces>
