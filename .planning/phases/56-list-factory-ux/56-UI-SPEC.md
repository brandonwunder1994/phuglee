# Phase 56 — UI Design Contract

>

## Spacing Scale

Declared values (must be exactly one of 4, 8, 16, 24, 32, 48, 64):

| Token | Value | Usage |
|-------|-------|--------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact / KPI inner gaps, toolbar control gaps |
| md | 16px | Default element spacing, stub-note / flash padding |
| lg | 24px | Section / panel padding, save-row gap |
| xl | 32px | Layout gaps between results blocks (save panel vs attach) |
| 2xl | 48px | Major section breaks (results → lists panel chrome only) |
| 3xl | 64px | Page-level only if already present — do not add |

### Exceptions

Existing `--space-xl: 2.5rem` (40px) and `--space-2xl: 4rem` (64px) in `tokens.css` stay as inherited Filter chrome. Phase 56 does not invent new spacing tokens. Light hierarchy polish may reuse existing classes only. (Same pattern as Phase 55: live `--space-2xl` maps to declared **3xl** 64px; live `--space-xl` at 40px is chrome inheritance only and is **not** a Phase 56 design value.)

**Do not** add 12px, 20px, or other off-scale gaps for workflow strip / flash / empty-state polish.

## Focal Point (Visual Hierarchy)

### Post-process (results panel has kept rows)

| Priority | Anchor | Role |
|----------|--------|------|
| **1 — Primary** | `#bridge-save-list` (“Save list”) | Undisputed post-process CTA — orange primary; operator lands here after process (and after Train when admin) |
| **2 — Secondary (context)** | Results KPIs + table | Informational; do not out-compete Save with peer primary buttons |
| **3 — Tertiary reassurance** | `#bridge-stub-note` | Independence / discard notes only — lighter weight than Save |
| **4 — De-emphasized** | `#bridge-export-csv` (renamed Preview CSV) | Ghost/secondary only; never primary class; never orange |
| **5 — Optional** | Attach panel | Form Forge KPI only; ghost/secondary |

### Post-save (Saved lists panel)

| Priority | Anchor | Role |
|----------|--------|------|
| **1 — Primary bulk** | `#bridge-download-all-csv` (“Download all (CSV)”) | Orange primary when toolbar visible (lists exist) |
| **2 — Primary per-list** | Per-row Download CSV (existing list actions) | Durable single-list handoff |
| **3 — Secondary bulk** | `#bridge-download-all-xlsx` | Ghost/secondary format option |
| **4 — Feedback** | `#bridge-lists-flash` | Success + next step (next city **or** download path) |
| **5 — Destructive** | Clear all / delete one | Danger-styled; always confirm |

### Forbidden hierarchy

- Never introduce Send/Push/Import to Analyze as any priority level
- Never promote Preview CSV or Attach above Save list
- Never make both Save list and Export/Preview share identical primary visual weight

|---------|------|

| Primary CTAs (post-save) | **Download all (CSV)**; per-list **Download** (CSV / XLSX labels as today) |
| Secondary bulk download | **Download all (XLSX)** |

| Forbidden CTAs | **Never show:** “Send to Analyze”, “Push to Analyze”, “Import to Analyzer”, “Open in Analyze”, “Push to Analyzer”, or any control that POSTs `/api/bridge-import-records` or calls push helpers |

### Teaching pack — Process → (Train) → Save → Download (LIST-03)

| Surface | Copy (locked target) |
|---------|----------------------|
| Hero lead (tighten) | Keep baseline intent: Filter stages multi-city lists for **download** and enrichment; **Analyze stays separate** — import there only after skip-trace. May expand one short clause for Save if needed without bloating. |
| Results heading | Keep **Results & save** (already teaches save). Optional subtitle not required if stub + save lead cover loop. |
| Process stub (clean / no discards of interest) | `Processing complete. Save the list below — nothing was sent to Analyze.` (Phase 55 lock — keep) |
| Process stub (discards present) | `{N} row(s) discarded.` + optional distress / review fragments — **omit** Analyze-hidden fragment when `alreadyImported === 0` |
| Save panel heading | **Save filtered list** (keep) |
| Save panel lead | `Store this list so you can process more cities. When ready, download from Saved lists for external enrichment. Nothing is sent to Analyze.` (tighten of live lead — explicit Download step) |
| Save empty guard | `Process a file with kept rows before saving.` (keep) |
| Save success flash | `Saved “{name}”. Upload the next city when ready — or download from Saved lists for enrichment.` |
| Lists panel lead | `Master staging: lists stay until you delete them. Download one or all for skip-trace / enrichment, then import into Analyze manually.` |
| Lists empty heading + body | `No saved lists yet. Process a city file, Train if needed (admin), click Save list, then Download one or all for external enrichment. Import into Analyze only after skip-trace.` |
| Train panel (admin) — optional one line in status or section lead | After decision success: `Decision saved to brain. Save list below when this city is ready.` |
| Train-before-Save soft confirm | See Interaction — soft confirm copy |
| Dirty-guard confirm | See Interaction — process clobber copy |

### Workflow strip (discretion — light touch)

Prefer **one** compact teaching line on results or lists (not a full pipeline renumber):

| Placement | Copy |
|-----------|------|
| Optional results workflow line (if added) | `Workflow: Process → (Train) → Save list → Download → enrich outside → manual Analyze import` |
| Pipeline steps | Keep 4 steps (City → Type → Upload → Results). **Do not** renumber entire pipeline to add Save/Download as step 5 this phase. |

### Empty / error / destructive

| Element | Copy |
|---------|------|
| Empty lists (above) | Full LIST-03 loop — not the short Phase 55 “Process… Save list” only |
| No lists download guard | `No saved lists to download yet.` (existing) |
| Generic process/save error | Server `error` + actionable next step via existing `fetchJson` / `showError` / `setSaveStatus` |
| `NO_USABLE_ROWS` | Phase 55 honesty: prefer distress / address / duplicate breakdown; **do not** lead with “already in Analyze” when filter off and counts are 0 |
| Destructive: delete one list | Existing confirm (keep strong; do not remove) |
| Destructive: clear all lists | Count-based confirm **required** (keep). Optional upgrade: typed `CLEAR` if cheap — not required for LIST success |
| Destructive confirmation pattern | Native `window.confirm` (or existing dialog); never silent wipe |

### Independence strings (inherit Phase 55 — do not regress)

| Allowed | Forbidden |
|---------|-----------|
| “nothing was sent to Analyze” (reassurance) | “sent to Analyze”, “queued for Analyze”, “pushed” as product path |
| “import into Analyze only after…” (manual, after enrich) | Implying Filter writes Analyze or auto-import |
| Positive `alreadyImported` note only when count > 0 under opt-in | Showing hidden/Analyze-index vanity when count is 0 |

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Process complete | Show results + Save panel for kept rows; never auto-navigate to Analyze; never auto-POST import; never auto-save list store |
| **Save list** | Explicit user click only on `#bridge-save-list`; POSTs `/api/bridge/lists`; success → `loadSavedLists` + `resetImportAreaAfterSave`; flash teaches next city **and** download path |
| **Download one** | From Saved lists row actions only (durable store); marks downloaded status as today |
| **Download all** | Toolbar `#bridge-download-all-csv` / xlsx; only when lists exist |
| **Preview CSV** | Client export of current `lastResult` only; no list store write; no downloaded status |
| **Dirty-guard (LIST-02 seam)** | On `processUpload()` entry: if `lastResult?.rows?.length > 0`, **confirm** before process. Cancel aborts process. Confirm proceeds and overwrites working set. **Do not** auto-save to avoid the dialog. |
| Dirty-guard copy | `You have {N} kept row(s) that are not saved yet.\n\nProcess a new file anyway? Unsaved work (including any Train decisions) will be lost.` Buttons: native OK/Cancel (OK = process and discard) |
| Optional beforeunload | Discretionary: when unsaved kept rows exist, may set `beforeunload` — not required for LIST success |
| **Soft Train-before-Save (admin only)** | On Save click: if `isBridgeAdmin()` and Train groups are still visible / Train mode active with open groups, **soft confirm** — not a hard block. Customers never see this. |
| Soft warn copy | `{N} Train group(s) are still visible. Save this list now?\n\nTip: Finish Approve/Deny first so this download matches your decisions.` Cancel returns to Train; OK continues save |
| Post-Train decision status | Soft nudge to Save list when city ready — brain already updated; list store only on Save |
| Attach | Optional Form Forge only; does not send leads to Analyze (existing lead) |
| Clear all / delete | Operator-only; process/restart/deploy never clear; agents never wipe `filter-lists` |
| Forbidden control | Any clickable that would push/import to Analyze — **not present**; static-ban tested |
| Opt-in import filter | Engine flag only; **no** Filter UI toggle this phase |

### Working-set model (must preserve)

```
lastResult (client memory only)
  processUpload → overwrites (after dirty-guard)
  Train decision → mutates rows / groups / stats
  Save → POST /api/bridge/lists (snapshot) → resetImportAreaAfterSave
  Download → from list store (durable), not lastResult
```
