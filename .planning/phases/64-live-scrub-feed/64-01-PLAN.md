### Pool mapping (must implement)

| Source | Status key | Address field |
|--------|------------|---------------|
| `data.rows[]` | `kept` | `streetAddress` (skip empty) |
| `data.notDistressedRows[]` | `no-distress` | `streetAddress` (skip empty) |
| `data.discarded[]` where reason is already-imported | `already-in-Analyze` | `rawPreview` (reason-only OK if empty preview — do **not** invent streets) |
| other `data.discarded[]` | `discarded` | `rawPreview` or reason fallback |

Already-imported detection: reason equals `already_imported` **or** matches `/already imported/i` (label: `Already imported in Analyze` from DISCARD_REASONS).

### Stratified sample + interleave

1. Build four pools as above.
2. Target mix of **cap** (not full file): ~40% kept / ~30% no-distress / ~20% discarded / ~10% already-in-Analyze; redistribute empty pools so remaining buckets fill the cap.
3. Round-robin interleave so feed is not “all kept then all kills.”
4. `events.length ≤ cap` (default 32).
5. `summary` from `stats.kept`, `stats.noDistress`, `stats.discarded`, `stats.alreadyImported` (coerce missing → 0). Optional `totalParsed` from stats.
6. `remainderByStatus[status] = max(0, statsBucketTotal − countShownForStatus)`. Prefer stats totals over pool length (FN list may be truncated server-side).
7. **Never** synthesize street strings. Empty pools → fewer events (OK). Empty `data` → `{ events: [], summary: zeros, remainderByStatus: zeros }`.
