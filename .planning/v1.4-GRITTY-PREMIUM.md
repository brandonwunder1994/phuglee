## North star

Not “dark SaaS with grain.”  
**Proof-first · work-first · one story spine · same film as the duck hero.**

If a surface still feels like a launcher menu, marketing feature grid, or empty hub — **it fails**.

---

## Quality bar (raised)

| Pillar | Meaning |
|--------|---------|
| **Proof-first** | Real counts, real coverage, same-address stories, distress imagery. Demo data only when labeled as illustrative of the product UI. |
| **Work-first** | Fewer hubs. User should feel “already in the job,” not choosing tiles. |
| **One story spine** | Home tells Collect→Filter→Analyze once. How It Works is a **slim playbook**, not a second landing page. |
| **Live when possible** | Dashboard mission CTA + status from real health/coverage. Coverage close uses real city/state counts. |
| **Restraint** | Grit from photo, type, voice, hierarchy — not grain wallpaper on every box. |

---

## Style DO / DON’T

### DO
- Distressed house atmosphere on marketing + key heroes
- Asymmetry (dominant panel + supporting scrap)
- Ops voice: Hit the Clerk · Scrub the Mess · Rank & Dial
- One fire CTA per view (`--phuglee-orange`)
- Anton display + cream ink on black
- Full duck only at hero / peak moments

### DON’T
- Equal icon+title+desc card grids
- Peer tile hubs (two equal cards as the whole page)
- macOS traffic-light mock chrome
- Wireframe bar mockups
- Three product windows pretending to be a “story”
- Corporate H1s (“Request Your Data”, “#1 Source for…”)
- Gradient text if avoidable (prefer solid cream; existing brand gradient titles OK if already system-wide)
- Infinite empty void around two tiles
