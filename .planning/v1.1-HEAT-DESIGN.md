# v1.1 Unified Heat Design — Design Specification

**Milestone:** v1.1  
**Reference:** Command Hub (`public/heat.html`, `public/css/hub.css`)  
**Date:** 2026-07-06  
**Status:** Approved

---

## Problem

Distress OS Command Hub uses a cohesive "Heat" aesthetic (warm browns, ember orange, Anton/Outfit), but Form Forge (gold stamp-theme) and Property Analyzer (Aerial Command amber/Fraunces) look like separate products. Navigation is fragmented — only shell pages share the Distress OS nav bar.

## Solution

1. **Heat design system** — Command Hub tokens become canonical across all apps
2. **Global nav menu** — same navigation bar on every page with links to all Forge subpages + Analyzer
3. **Visual reskin only** — zero backend or JS behavior changes

## What v1.8 Aerial Command Means (user clarification)

Property Analyzer **v1.8** introduced the "Aerial Command" look:
- **Fonts:** Fraunces (editorial serif headlines) + DM Sans
- **Colors:** Amber gold `#e8a838` accents on dark blue-gray `#0c0e12`
- **Feel:** Satellite intelligence console, photography-forward

**v1.1 replaces this** with Command Hub Heat:
- **Fonts:** Anton + Outfit
- **Colors:** Ember orange `#e85d04` + flame `#ff6b1a`
- **Feel:** "You're in the heat" — warm, urgent, unified with Distress OS

Functionality is unchanged — only the visual skin swaps.

## Constraints

## Success Criteria

1. User can navigate from any page to any other page via global nav
2. Visual side-by-side: all apps look like one product family
3. No regressions in Form Forge workflows or Analyzer scan/review/export
