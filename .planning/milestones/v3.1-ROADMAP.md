# Roadmap: v3.1 Analyzer Scan Desk (Linear)

> **Active milestone:** v3.1  
> **Audit:** `.planning/debug/2026-07-12-analyzer-scan-desk-audit.md`  
> **Phase numbers:** 82–88 (continues from v3.0 phase 81)

## Overview

Make Analyze a **linear scan desk**: upload a list, scan it, get four buckets (Distressed / Well Maintained / Land / Blocked), review those groups. Fix upload crashes on large sessions and stop marketing the residual “Needs Review” KPI as the primary story.

## Phases

- [ ] **Phase 82: Stabilize** — Process health + lean scan-queue save (STAB-01–03)
- [ ] **Phase 83: KPI Truth** — Four-bucket heroes + live KPIs (KPI-01–03)
- [ ] **Phase 84: Upload Isolation** — New batch queue + stamp + no full rewrite (UP-01–04)
- [ ] **Phase 85: Scan Resume** — Pending/this-list accuracy (SCAN-01–03)
- [ ] **Phase 86: Review Desk** — Review Leads 4 buckets + residual (REV-01–03)
- [ ] **Phase 87: History Secondary** — Collapse historical; linear hero (HIST-01–02)
- [ ] **Phase 88: QA Lock** — Tests + verify-live (QA-01–03)

## Success criteria (milestone)

1. Operator can upload a new CSV with a ~10k historical session without tab crash
2. Start Scan only processes unscanned addresses on that list
3. KPIs show Distressed / WM / Land / Blocked / Scanned
4. Review Leads opens each of the four groups
5. Local + prod `/analyzer` healthy (or local verified when prod ops pending)
