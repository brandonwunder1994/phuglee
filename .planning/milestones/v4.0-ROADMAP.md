# Roadmap: v4.0 The Vault (Leads Platform)

> **Active milestone:** v4.0 (starts after v3.1 Analyzer Scan Desk)  
> **Spec:** `docs/superpowers/specs/2026-07-13-leads-platform-design.md`  
> **Plan:** `docs/superpowers/plans/2026-07-13-leads-platform.md`  
> **Phase numbers:** 89–97

## Overview

Transform `/vault` from a gated marketing preview into a **premium curated leads database** for Max-plan members. Three lead types: Distressed (post-review), Well Maintained, Land/Vacant Lot. Fed by the existing Filter → Analyze pipeline.

## Phases

- [ ] **Phase 89: Foundation** — Lead schema, fixture catalog, read API (`lib/leads-platform/`)
- [ ] **Phase 90: Vault shell** — Live app mount replaces frosted mock for Max users
- [ ] **Phase 91: Filter engine** — Search, geo, stackable motivation signals
- [ ] **Phase 92: Detail drawer** — Expandable lead dossier (photos, comps, owner, financials)
- [ ] **Phase 93: Scoring UX** — Priority rank, signal chips, default sort
- [ ] **Phase 94: Operator tools** — Favorites, notes, export
- [ ] **Phase 95: Publish pipe** — Analyze/Filter → catalog ingest
- [ ] **Phase 96: Plan gates** — Max billing UX, shell nav, honest Pro gate
- [ ] **Phase 97: QA lock** — Tests, a11y, verify-live

## Success criteria

1. Max user sees real catalog (or honest empty state) — never fake frosted rows as live data
2. Filter + search surfaces top-priority leads via signal stacking
3. Lead dossier is dial-ready (owner, phone, signals, financials when available)
4. Distressed leads only appear after `reviewStatus === 'approved'`
5. UI matches Phuglee brand (dark earth, cream, gold heat)
6. `npm test` green; `verify-live.ps1` exit 0

## Out of scope (v4.0)

Stripe billing, map view, CRM sync, React rewrite
