## 6. Logo SVG Integration Strategy

**File:** `public/images/phuglee-logo.svg` (408×612, multi-path illustration)

| Usage | Treatment |
|-------|-------------|
| **Home hero** | Full mascot, scale 1.4–1.6, subtle float animation |
| **Nav** | `phuglee-text-logo.svg` cream wordmark |
| **Background pattern** | Extract 3–5 path groups → `phuglee-pattern.svg` tile |
| **Empty states** | Mascot watermark at 8% opacity |
| **Favicon / OG** | Simplified mark from flame/house cluster |

**Implementation path (locked for M4):**

1. **Phase 22:** Create `public/js/phuglee-logo.js` — vanilla SVG injector with `data-phuglee-logo` attributes (no React required for current stack)
2. **Optional Phase 30+:** React wrapper if shell migrates to Vite/React

**Performance:** Single SVG fetch, `cache-control`, no duplicate inline paths, `prefers-reduced-motion` disables float.

## 7. Motion System

| Interaction | Animation |
|-------------|-----------|
| Page enter | Stagger fade-up 0.6s, children 80ms delay |
| CTA hover | translateY(-2px) + glow intensify |
| Panel hover | translateX(4px) + border orange |
| Modal | scale 0.96→1 + opacity |
| Nav link active | underline grow from center |

**Stack decision (M4 Phase 22):** CSS `@keyframes` + `IntersectionObserver` stagger in vanilla JS. Framer Motion deferred unless shell migrates to React (documented in Phase 30 research spike).
